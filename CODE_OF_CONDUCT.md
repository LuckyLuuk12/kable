# Code of Conduct

## Purpose

This project is focused on building quality software. We expect contributors to maintain a professional environment where technical discussions can happen productively.

## Standards

### Expected Behavior

- **Be professional** in issues, pull requests, and discussions
- **Focus on technical merit** - Critique ideas and code, not people
- **Accept constructive feedback** - Code reviews are about improving the code
- **Stay on topic** - Keep discussions relevant to the project

### Unacceptable Behavior

The following will result in warnings, removal of contributions, or bans:

- **Harassment** - Unwelcome comments, personal attacks, or intimidation
- **Spam** - Excessive promotion, off-topic content, or noise
- **Doxxing** - Publishing private information without permission
- **Trolling** - Deliberately inflammatory or disruptive behavior
- **Illegal activity** - Any illegal content or activity

## Scope

This Code of Conduct applies to:
- GitHub issues, pull requests, and discussions
- Project communication channels
- Representing the project in public spaces

## Enforcement

Project maintainers have the right to remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct.

Violations can be reported to the project maintainers. All reports will be reviewed and may result in:
- Warning
- Temporary restriction from participation
- Permanent ban from the project

Decisions on enforcement are at the discretion of project maintainers and are final.

## Bottom Line

We're here to build software. Treat others professionally, follow the law, and focus on the work. That's all we ask.
